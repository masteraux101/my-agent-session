# Quantum Framework Documentation Review (2025/2026)

## Qiskit (IBM)
- **Latest Doc URL:** https://docs.quantum.ibm.com/
- **Readability:** Very High. IBM has unified their documentation, providing a seamless path from learning basics to advanced hardware execution.
- **GitHub Status:** Highly active. Transitioning to `qiskit` 1.x has resolved many legacy issues but created some breaking changes.

## Cirq (Google)
- **Latest Doc URL:** https://quantumai.google/cirq
- **Readability:** Medium-High. Very research-oriented. Documentation is thorough but can be dense for beginners.
- **GitHub Status:** Active, but more focused on specific hardware (Sycamore) and NISQ research.

## Q# (Microsoft)
- **Latest Doc URL:** https://learn.microsoft.com/en-us/azure/quantum/
- **Readability:** High (Improved). The "Modern Q#" shift removed the heavy .NET dependency for many tasks, making it much more accessible via VS Code and Python.
- **GitHub Status:** Moderate. The move to the new QDK (written in Rust) has revitalized the repository.
